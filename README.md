# Study Repository

Универсальный репозиторий для домашних работ и проектов по разным предметам.

## Текущая структура

- `common/` — общие правила, шаблоны и скилы.
  - `common/rules/` — кросс-предметные правила выполнения и оформления.
  - `common/skills/` — кросс-предметные agent skills.
  - `common/templates/` — шаблоны, методические документы, требования по оформлению.
- `subjects/` — предметные каталоги.
  - `subjects/omm/` — материалы по ОММ.
    - `resources/` — методички, условия задач.
    - `examples/` — примеры чужих решений/отчетов (как референс, не как источник истины).
    - `tasks/` — собственные постановки/входы по вариантам.
    - `reports/` — итоговые отчеты.
    - `skills/` — скилы именно для ОММ.
    - `rules/` — правила именно для ОММ.

## Политика языков

- Базовый язык: **Go**.
- Для численных методов и научной визуализации можно (и часто лучше) использовать **Python** (NumPy/SciPy/Matplotlib).
- **MATLAB/Octave** — если это требуется преподавателем/методичкой или нужно воспроизвести формат конкретного курса.

## Почему графики чаще «похожи» в ОММ

Проверенная практическая причина: учебные и научные отчеты часто строят в MATLAB или в Python Matplotlib; Matplotlib изначально предоставляет интерфейс `pyplot`, который работает «как MATLAB».
Поэтому стиль графиков у разных людей выглядит схожим.
