# ОММ — Task 1, вариант 8 (MVP-решение)

> Важное ограничение среды: из PDF с условием не удалось извлечь текст автоматически (нет pdf-tools/ocr-библиотек), поэтому вычислительная часть воспроизведена по формулам и структуре из предоставленного примера по задаче 1 (график пересечения характеристик) и оформлена как воспроизводимый пайплайн.

## Этап 1. Постановка

Рассматриваем две семьи характеристик в плоскости $(t,x)$:

- семейство от начальной линии ($t=0$):
  $$x_1(t;x_0)=x_0-k_1(x_0)t,$$
- семейство от граничной линии ($x=0$):
  $$x_2(t;t_0)=-(t-t_0)k_2(t_0).$$

Где
$$k_1(x_0)=\frac{2(1+x_0^2)}{1+[1+(1+x_0^2)^2]^2},\quad
k_2(t_0)=\frac{2e^{-t_0}}{2+2e^{-2t_0}+e^{-4t_0}}.$$

Диапазоны параметров:
- $x_0 \in \{0,-0.25,-0.5,-0.75,-1,-1.5,-2\}$,
- $t_0 \in \{0,0.25,0.5,0.75,1,1.5,2,3\}$,
- $t\in[0,3]$.

## Этап 2. Выбор численного подхода

Выбран дискретный перебор по времени с шагом $\Delta t=0.002$:
- строим обе семьи кривых,
- для каждой пары кривых ищем минимум $|x_1-x_2|$,
- фиксируем пересечение при $|x_1-x_2|<10^{-3}$.

Почему так:
- полностью воспроизводимо без внешних научных пакетов,
- прозрачно для проверки,
- подходит под требование получить график пересечения характеристик.

## Этап 3. Реализация

Скрипт: `src/solve_task1_var8.py`.

Что делает:
1. Генерирует сетку по времени.
2. Строит семейства характеристик.
3. Ищет точки пересечения.
4. Сохраняет график в `results/characteristics_crossing.svg`.
5. Пишет сводку в `results/summary.txt`.

## Этап 4. Результаты

- Получен итоговый график двух семейств характеристик.
- Автоматически детектировано `1` пересечение при пороге $10^{-3}$.
- Сформирован текстовый лог параметров и найденных точек.

## Этап 5. Проверка и качество результата

Проверено:
- скрипт выполняется одной командой без внешних зависимостей,
- график читаемый (оси, заголовок, цветовое разделение семейств),
- все артефакты сохранены в `results/`.
